Commenting Your Code

Last but not least we have comments in Python. If you know anything about programming you'll know that comments are essential, as without them, someone else reading your code may not have any clue what it's supposed to do!.

In Python, there are two types of comments - single and multi-line comments. Single-line comments are red and are denoted by a hash. Multi-line comments are green, like strings, and are denoted by three quotes on each end (single or double).

There are a few conventions which govern Python comments - these are available to read in PEP8. It is strongly recommended you read these. There is also an online checker/validator which you can run your code through to make sure it is following the PEP8 coventions.
