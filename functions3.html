FUNCTIONS

Global vs Local Variables

If you want a variable, constant or array to be available to every function, it should be created in the main routine. Such a variable is called a 'global' variable. All the variables/constants/arrays you have created so far have been global.

Any variable created inside a function is considered 'local' to that function. A local variable cannot be used outside of its function.

Scope

Local variables and global variables would usually have different names, but it is possible for them to have the same name. This raises for the first time the concept of scope, which is about where a variable is accessible within the structure of the program.

In the following example the name “x” belongs to both a local variable in function_one, as well as the global variable in function_two and the main routine. 

# function definitions 

def function_one():
	x = 8
	print("x is", x)

def function_two():
	print("x is", x)

# main routine

x = 5
print("x is", x)
function_one()
function_two()
