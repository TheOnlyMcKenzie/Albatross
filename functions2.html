Docstrings

As we have said before, commenting your code is an essential part of programming, as code is often read more than it is written.

Python documentation strings (docstrings) provide a way of associating documentation with Python modules, functions, classes and methods.

Docstrings are written like multi-line comments (even though they can be as short as one line). They appear as the first statement in a function's definition to describe what the function does. All functions should have a docstring.

Example of a function with a single-line docstring: 

def add_1(num):
	"""Adds 1 and prints."""
	print(num + 1)

Example of a function with a multi-line docstring:

def add_5(num):
	"""Adds 5
	   returns it as a string
	"""
	return(str(num + 5))
